# feat: 元策 V1 产品完整版本蓝图

## 当前结论

元策 V1 已经具备“单体可部署、统一 `/web`、SQLite 持久化、RBAC、项目与工作项基础闭环”的主体骨架。下一阶段不再扩张为复杂禅道复刻，而是把现有能力补成团队日常可连续使用的产品流：

- 项目内直接创建需求、任务、Bug。
- 工作项可编辑核心字段，不只做状态和评论。
- 工作项列表具备基础筛选。
- 文件附件接入现有对象存储和文件元数据底座。
- 项目动态覆盖关键业务动作。
- 系统管理保持轻量，服务于用户、权限、对象存储和审计。

## 已完成能力

- 统一入口：
  - `/` 跳转 `/web`。
  - 不提供独立 `/admin`。
  - `/api` 作为 JSON 接口边界。
- 认证与安全：
  - 首次管理员初始化。
  - 开发测试 local-admin seed。
  - 登录、退出、session 失效。
  - Web 写操作 CSRF。
- 权限：
  - RBAC 权限点、角色、角色授权。
  - 系统管理菜单和页面权限控制。
  - 项目成员关系控制项目数据范围。
- 项目协作：
  - 项目列表、创建、详情。
  - 项目成员添加和移除。
  - 需求、任务、Bug 统一工作项模型。
  - 工作项列表、详情、状态流转、评论。
  - 项目活动。
  - 我的页面和全局搜索。
- 系统管理：
  - 用户管理。
  - 角色权限。
  - 对象存储配置。
  - 审计日志。
- 工程规范：
  - 显式迁移。
  - SQLite 为唯一真实数据源。
  - 不引入 Redis。
  - HTML/CSS/JS/htmx 打包进 Rust 二进制。
  - 后续默认在 `main` 分支开发、提交、推送。

## V1 完整版本剩余主线

### P0：项目内工作项创建闭环

项目详情页是团队使用频率最高的入口，必须支持在当前项目内直接创建需求、任务、Bug，而不是跳到跨项目列表再选择项目。

交互：
- 项目详情页右上角“新建工作项”跳到页内创建表单。
- 创建表单默认锁定当前项目。
- 用户选择类型、优先级，填写标题和描述。
- 创建成功后回到当前项目详情。

验收：
- 项目成员可在已加入项目内创建三类工作项。
- 非项目成员不能绕过表单创建该项目工作项。
- 创建后项目详情对应分类和项目动态立即可见。

### P1：工作项编辑

工作项详情页需要支持修改日常字段：
- 标题。
- 描述。
- 优先级。
- 负责人。
- 状态。

验收：
- Web 和 API 复用同一 domain 校验。
- 修改写入项目动态。
- 负责人只能选择项目成员。

当前状态：已完成。Web 工作项详情页支持编辑核心字段；API `PATCH /api/v1/work-items/:item_key` 复用同一 domain 更新逻辑；负责人校验为已启用项目成员。

### P1：列表筛选

跨项目需求、任务、Bug 列表需要支持基础筛选：
- 关键词。
- 状态。
- 优先级。
- 所属项目。
- 负责人。

验收：
- 页面使用 GET 查询参数，便于复制链接。
- 普通成员只能看到自己项目范围内的数据。
- API 列表参数与 Web 查询语义保持一致。

当前状态：已完成。需求、任务、Bug 列表页支持 GET 参数筛选；API `GET /api/v1/work-items` 支持同名筛选参数，并继续受项目成员数据范围约束。

### P1：附件最小业务入口

文件底座已存在，下一步需要接入项目和工作项：
- 项目附件。
- 工作项附件。
- 评论附件后置。

第一版可先做“登记已上传文件 + 业务挂载”，签名上传下载再按 OpenDAL/OSS 能力补齐。

验收：
- 文件元数据记录 object key、bucket、大小、content type。
- 附件挂到项目或工作项。
- 项目成员可见，非成员不可见。
- 对象存储密钥不出现在页面、日志和响应中。

当前状态：已完成最小入口。项目详情和工作项详情支持登记已上传文件元数据并挂载附件；附件复用 active 对象存储配置和 `file_objects` / `file_attachments` 底座。签名上传、下载和评论附件后续增强。

### P2：项目动态完善

项目动态应覆盖：
- 创建项目。
- 添加 / 移除成员。
- 创建工作项。
- 更新工作项字段。
- 状态流转。
- 新增评论。
- 添加附件。

验收：
- 项目详情动态能按时间展示最近协作动作。
- 动态写入和业务动作处于同一事务。

当前状态：已完成。创建项目、成员变更、创建工作项、编辑工作项、状态流转、评论和附件登记都会写入项目动态；核心业务动作在 domain 事务内写入动态。

## 产品边界

V1 暂不做：
- 测试用例、测试计划、发布管理。
- 甘特图、燃尽图、复杂报表。
- 多租户、多 portal、多后台。
- Redis、MQ、分布式锁、多实例一致性。
- React/Vue/Tailwind/Ant Design 独立前端。
- 多对象存储供应商。

## 当前执行结果

已按本蓝图完成当前 V1 产品闭环：

1. 已实现 P0 项目内工作项创建闭环。
2. 已实现 P1 工作项核心字段编辑。
3. 已实现 P1 工作项列表和 API 基础筛选。
4. 已实现 P1 项目 / 工作项附件最小入口。
5. 已实现 P2 项目动态覆盖和关键业务动作事务内写入。

## 后续增强

以下事项不阻塞当前 V1 闭环，后续可按优先级继续迭代：

- 对象存储签名上传和签名下载页面/API。
- 评论附件。
- 附件删除和上传完成状态回写。
- 更丰富的项目动态筛选。
- 工作项批量操作和更细粒度列表分页。
