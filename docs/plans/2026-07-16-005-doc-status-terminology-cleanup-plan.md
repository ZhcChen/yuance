# docs: 历史计划文档状态术语清理

## Overview

前几轮已经完成运行时代码、Web 展示层和 OpenAPI 契约里的状态统计命名统一，但部分历史计划文档里仍保留“开放工作项”“开放状态”“开放项”这类旧业务表述。

这些内容虽然不影响程序运行，但会让后续阅读计划文档的人继续接触到已经被淘汰的语义。本轮只做文档层清理，把业务描述统一成更直接的：

- 待处理 / 进行中 / 待确认
- 活跃未完结项
- 待处理统计口径

## Scope

- 修改历史计划文档中的旧业务表述
- 保留代码标识、历史字段名和迁移背景说明
- 不修改任何运行时代码

## Non-Goals

- 不改 `open` 状态码本身
- 不重写整份历史计划，只做术语收口
- 不改“开放平台”“开放入口”这类与工作项状态无关的表述

## Implementation Units

- [x] Unit 1: 盘点需要替换的历史计划文档
- [x] Unit 2: 更新旧业务表述
- [x] Unit 3: 扫描残留并完成校验
- [x] Unit 4: 提交并推送

## Verification Focus

- 历史计划文档中的业务状态表述不再传播“开放项”概念
- 与迁移相关的旧字段名、旧函数名说明仍保留必要上下文
- 本轮 diff 仅限文档文件
