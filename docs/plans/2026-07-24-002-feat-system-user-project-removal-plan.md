# 系统用户页项目移除增强计划

## 目标

- 在系统用户管理的“分配项目”弹窗中，直接支持移除用户已加入的项目。
- 复用现有项目成员移除规则，避免系统页与项目详情页出现两套不同约束。
- 对存在待处理工作项的项目成员，在 UI 上直接给出不可移除提示。

## 范围

- 系统用户页已加入项目数据补充“待处理数量 / 可否移除”信息。
- 新增系统用户页项目移除提交入口。
- 弹窗中的“当前已加入项目”区域增加移除按钮与限制提示。
- 补充系统管理回归测试。

## 关键决策

- 待处理数量口径直接复用现有成员移除规则：`done / closed / resolved / verified / cancelled` 之外均视为阻塞移除。
- 当前已加入项目中，如该用户在对应项目仍有待处理工作项，则移除按钮禁用并显示原因。
- 项目负责人移除继续交给后端原有校验兜底，不在本轮前端额外做项目负责人识别。

- [x] T2. 新增系统用户页项目移除路由与处理
- [x] T3. 调整弹窗当前项目列表 UI 与移除交互
- [x] T4. 补充测试并执行校验
