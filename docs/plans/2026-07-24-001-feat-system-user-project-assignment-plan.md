# 系统用户页直接分配项目实施计划

## 目标

- 在系统管理的用户列表中增加“直接分配项目”能力。
- 管理员无需逐个切换项目，即可把一个用户批量加入多个项目。
- 分配后继续沿用现有项目成员体系与成员角色体系，不引入第二套关系模型。

## 范围

- 用户管理列表补充分配项目入口、项目概览列与批量分配弹窗。
- 后端新增系统用户页批量分配项目提交入口。
- 复用现有项目成员写入逻辑与审计记录。
- 补充系统管理最小回归测试。

## 关键决策

- 只做“批量加入项目”，本轮不在系统用户页补“批量移除项目”。
- 分配项目时统一选择一个项目角色，应用到本次选中的所有项目。
- 只展示当前允许写入的项目作为可分配候选，避免把已暂停、已完成、已归档项目暴露成可操作项。
- 页面内使用单例弹窗承载分配交互，避免每个用户重复渲染整套项目候选列表。

## 任务拆解

- [x] T1. 补充系统用户页数据装配
  - 为用户列表追加已分配项目摘要。
  - 加载系统级可分配项目候选。
  - 增加“是否可管理用户项目”的权限判断。

- [x] T2. 新增批量分配项目提交入口
  - 增加 `/web/system/users/{username}/projects` POST 路由。
  - 解析多选 `project_key` 表单并复用 `projects::add_project_member`。
  - 保留分页参数并补充审计日志。

- [x] T3. 实现前端弹窗与交互
  - 用户列表增加项目列与“分配项目”按钮。
  - 弹窗展示当前已分配项目、可选项目搜索、多选与角色选择。
  - 打开弹窗时动态过滤掉目标用户已加入的项目。

- [x] T4. 测试与校验
  - 覆盖页面渲染与批量分配提交。
  - 执行 `cargo test -p yuance-api system_management_flow`、`cargo check`、`git diff --check`。

## 验证

- 用户管理页能看到项目分配入口与当前项目概览。
- 选中多个项目后可一次性完成加入，并返回原分页位置。
- 已经加入的项目不会在候选列表中重复出现。
