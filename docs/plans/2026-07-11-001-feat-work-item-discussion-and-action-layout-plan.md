# 工作项讨论串与操作布局实施计划

## Task List

- [x] **T1. 评论回复数据模型**
  - 新增 `parent_comment_id` 自关联迁移与索引。
  - 扩展领域模型、Web/API 请求和响应，校验父评论属于同一工作项且不是流程记录。
  - 将扁平评论组装为可递归渲染的讨论树。

- [x] **T2. 永久保留协作记录**
  - 移除工作项、评论和评论附件删除路由与页面入口。
  - 保留编辑评论能力和审计记录。
  - 明确展示发表时间与编辑时间。

- [x] **T3. 指派与关闭语义**
  - 指派弹窗默认保持当前状态，允许只变更处理人并反复转派。
  - 流程记录突出操作者、原处理人和下一处理人。
  - 关闭动作根据需求、任务、Bug 显示类型化文案；关闭后提供重新打开。

- [x] **T4. 重构工作项详情布局**
  - 左侧整合正文、只读附件、流程和讨论串。
  - 右侧固定操作栏提供编辑、指派、关闭/重新打开。
  - 移除原字段面板、顶部按钮堆和底部独立上传入口。

- [x] **T5. 讨论附件与内联进度**
  - 新建共享讨论编辑器，支持多文件本地预览和移除。
  - 发表或回复时先创建评论，再逐个直传附件。
  - 上传进度固定显示在当前编辑器内，失败可重试。

- [x] **T6. 全站滚动条与视觉回归**
  - 全局隐藏标准和 WebKit 滚动条，保留滚动能力。
  - 完成桌面和移动端详情、回复、指派、关闭和媒体预览检查。
  - 更新迁移、领域、路由、模板和浏览器测试。
