# 系统用户页项目批量管理增强计划

## 目标

- 在系统用户管理的“分配项目”弹窗中，继续增强“当前已加入项目”区域。
- 支持搜索当前已加入项目、批量移除项目成员、直接调整该用户在项目内的成员角色。
- 保持系统用户页与项目详情页使用同一套项目成员规则，不引入旁路逻辑。

## 范围

- 已加入项目数据补充成员角色信息。
- 新增系统用户页项目批量移除入口。
- 新增系统用户页项目成员角色调整入口。
- 弹窗 UI 增加当前项目搜索、批量选择、批量移除和角色维护。
- 补充系统管理回归测试。

## 关键决策

- 当前已加入项目搜索支持按项目编码、项目名称、成员角色、项目状态过滤。
- 批量移除采用“先校验后执行”策略：只要任意目标项目不满足移除条件，则整批拒绝，避免部分成功、部分失败。
- 项目负责人仍不允许通过该弹窗调整角色或移除，继续要求在项目详情中转移负责人。
- 项目角色调整沿用既有角色枚举：`maintainer / member / viewer`。

## 任务拆解

- [x] T1. 扩展已加入项目数据装配
  - 按用户读取项目成员关系，并返回成员角色。
  - 结合待处理工作项统计，生成可移除 / 可调整角色状态。

- [x] T2. 增加系统用户页项目批量管理接口
  - 新增批量移除提交入口。
  - 新增单项目角色更新提交入口。
  - 保留分页参数并补审计日志。

- [x] T3. 调整弹窗当前项目区域交互
  - 增加当前项目搜索框。
  - 增加批量勾选与批量移除操作。
  - 每个项目卡片增加角色展示与角色保存动作。

- [x] T4. 补充测试与校验
  - 覆盖页面渲染、批量移除、阻塞场景、角色更新。
  - 执行 `cargo test -p yuance-api --test system_management_flow`、`cargo check`、`node --check api/static/app.js`、`git diff --check`。
