# 全站 UI 系统与工作项创建器升级

## Problem Frame

当前界面虽然已有基础颜色变量和部分共享样式，但按钮、输入框、下拉、标签、Tabs、弹窗和上传区分别演进，导致高度、文字垂直对齐、边界、圆角和动效不一致。工作项列表重复提供顶部导航已有的类型切换。工作项创建能力也不一致：只有 Bug 支持图文分组，每组只能上传一张图片，需求和任务无法在创建时附带说明附件。

## Requirements

**统一控件系统**
- R1. 按钮、输入框、文本域、选择器和图标按钮使用统一高度、字号、行高、圆角、边界、焦点、禁用和加载状态。
- R2. 所有普通选择器使用统一的自定义下拉面板，不依赖实验性浏览器样式；处理人、指派人、父需求等长选项支持搜索。
- R3. 工作项类型、优先级和状态标签使用稳定尺寸及明确的语义色，避免灰色胶囊堆叠。
- R4. 弹窗进入和退出具备一致的遮罩、淡入缩放动效，关闭按钮为圆形且图标严格居中。
- R5. 图片/视频查看器工具按钮使用圆形图标容器，所有图标保持视觉和几何居中。

**导航与 Tabs**
- R6. 顶部需求、任务、Bug 数量角标固定在导航项右上角，不参与文字行内排版。
- R7. 工作项列表移除重复的需求/任务/Bug 类型切换，只保留顶部全局导航。
- R8. 项目详情 Tabs 使用高对比的活动滑块，并在切换时平滑移动；字体、计数和容器配色与全站一致。
- R9. 顶部搜索按钮常驻主色；本地记录最近 5 个非空搜索词，输入框聚焦时提供可点击历史。

**工作项创建与附件**
- R10. 新建需求、任务和 Bug 均由当前顶部项目上下文确定归属，不显示只读项目字段。
- R11. 三类工作项均支持图文说明分组，每组可选择多个任意允许类型的附件。
- R12. 每组说明创建为工作项评论，多个附件关联同一评论并按文件显示上传进度、完成和失败状态。
- R13. 本地图片和视频可在上传前预览；已上传图片和视频可在详情页进入统一媒体查看器；普通文件显示清晰文件信息。
- R14. 文件类型与大小仍遵守服务端校验、签名直传、权限和 CSRF 边界。

**全站排版**
- R15. 筛选栏、表单操作区、列表元信息和工具栏在桌面及移动端保持稳定对齐，无横向溢出、文字偏移或控件高低不一。
- R16. 对所有现有 Web 功能页执行控件和布局巡检，修正与统一规范冲突的明显问题。

## Success Criteria

- 所有可见按钮与单行表单控件遵循统一尺寸，文字上下居中。
- 下拉触发器、选项面板、搜索、键盘导航、禁用态和表单提交均可用。
- 项目 Tabs 活动滑块清晰可辨并具有连续位移动画；弹窗开关有可感知过渡。
- 需求、任务、Bug 创建时均可添加多组说明和多文件，图片/视频可预览。
- 桌面和移动端关键页面没有重叠、溢出和明显对齐问题。

## Scope Boundaries

- 不引入新的前端框架或第三方组件库。
- 不改变工作项状态机、权限模型和对象存储签名方式。
- 搜索历史只保存在当前浏览器 localStorage，不同步到服务端。
- 视频预览只支持浏览器原生可播放格式，不做转码、封面抽帧或流媒体处理。

## Key Decisions

- 原生 `select` 继续作为表单数据源，自定义控件只负责展示和交互，避免改变后端契约。
- 图文创建流程复用现有工作项、评论和评论附件 API，不新增数据库结构。
- 控件规范落在现有 `api/static/app.css` 与 `api/static/app.js` 共享层，模板只声明语义属性。
