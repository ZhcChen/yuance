# 工作台项目推进与个人分析需求

## Problem Frame

工作台当前“我的工作项”与顶部需求、任务、Bug 导航重复，项目推进表又缺少个人待办和个人贡献视角。用户需要从项目列表快速进入自己的待处理工作，并查看自己在单个项目中的真实处理产出和协作情况。

## Requirements

**工作台**
- R1. 移除工作台“我的工作项”区域。
- R2. 项目推进表增加当前用户的待处理需求、任务、Bug 数量，并可跳转到对应列表。
- R3. 项目推进表增加“查看”操作，进入当前用户在该项目中的个人分析页。

**个人项目分析**
- R4. “已处理”仅统计当前用户亲自将工作项推进到完成、解决、验证或关闭状态的事件，不按当前负责人反推历史贡献。
- R5. 展示累计完成量及需求、任务、Bug 分布，当前待处理分布，近 30 日完成量。
- R6. 展示日均、单日峰值、月均、单月峰值；均值从用户加入项目到今天按自然日和自然月计算，包含零处理周期。
- R7. 展示评论/回复数量、流转次数、活跃天数及最近完成记录，并说明统计口径。
- R8. 页面只能查看当前用户有权访问的项目数据。

**工作项列表**
- R9. 需求、任务、Bug 列表移除顶部介绍 Hero，只保留统计区和数据列表。
- R10. 新建按钮移动到列表面板标题栏，并继续服从项目选择与权限控制。
- R11. 列表支持通过 URL 预置当前用户和开放状态，以承接工作台待处理快捷入口。

## Success Criteria

- 工作台不再出现重复的“我的工作项”。
- 每个项目可直接看到并进入个人待处理分类。
- 个人分析数据来源可解释，转派后不会把他人的完成量算给当前负责人。
- 三类工作项列表布局一致，新建入口位于列表区域。

## Scope Boundaries

- 本轮不建设团队排名、绩效评分或跨成员对比。
- 本轮不补算系统引入活动记录之前无法识别操作者的历史完成数据。
- 待处理只表示当前指派给用户且未进入终态的工作项。
