# 工作项讨论串与操作布局升级

## Problem Frame

需求、任务和 Bug 详情页目前把字段、快捷状态、评论、附件和流程记录分散在多个面板中，主次关系不清晰。评论是扁平列表，无法表达用户对用户的回复；独立附件区和上传弹窗也脱离讨论上下文。删除入口虽然使用软删除，仍不符合“所有协作记录永久保留”的产品语义。

## Requirements

- R1. 全站隐藏滚动条轨道，但鼠标、触控板、触屏和键盘滚动能力保持不变。
- R2. 工作项详情采用左右结构：左侧是标题、正文、附件和讨论；右侧是固定操作栏。
- R3. 右侧操作栏只承载编辑、指派/流转、关闭或重新打开等动作，不再重复字段面板。
- R4. 指派允许在状态不变时反复转派，并永久记录操作者、原处理人、下一处理人、状态和说明。
- R5. “关闭”按类型显示为“关闭需求”“关闭任务”“关闭 Bug”；不再出现“取消工作项”或删除工作项。
- R6. 评论支持回复任意普通评论，显示回复对象，并按讨论串嵌套展示。
- R7. 评论不可删除，只能由具备权限的用户编辑；原始发表时间始终显示，编辑后额外显示编辑时间。
- R8. 发表和回复均支持多附件；图片和视频可预览，其他文件可下载。
- R9. 文件选择只进入待发布状态，提交内容后才创建评论并逐个上传；进度显示在对应发表/回复编辑器内部。
- R10. 移除详情页底部独立“上传附件”操作；既有工作项附件继续只读展示，后续附件归属具体讨论内容。
- R11. 关闭后的工作项仍保留全部内容，可重新打开；关闭状态下仍允许有权限的成员继续讨论和转派。

## Success Criteria

- 用户能连续把同一工作项指派给不同项目成员，时间线明确显示“谁将谁指派给谁”。
- 任意普通评论都能展开回复编辑器，发布后出现在对应讨论串中。
- 页面不存在工作项、评论或附件删除按钮和可调用删除路由。
- 桌面端右侧操作栏滚动时保持可见，移动端自然回落为单列。
- 上传进度不会出现在页面底部或脱离当前编辑器。

## Scope Boundaries

- 不实现富文本编辑器、@通知、已读未读、实时推送和表情反应。
- 不物理删除历史软删除字段或旧数据。
- 不改变对象存储签名直传、安全权限和 CSRF 边界。
